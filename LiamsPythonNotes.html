    Boolean Logic¶

    Be careful when using boolean logic though, because sometimes it doesn't work the way it looks

    In [45]:
    weathergood = True
haveumbrella = False

if not haveumbrella or weathergood:
    print("You're gunna get wet")
else :
    print('Ready to walk')

    You're gunna get wet

    In this example, we get the wrong answer, because the not statement applies only to haveumbrella, not the weathergood.

    In most cases, like in language. the best way to do this is to remove any unneccessary negatives.

    Bytes¶

    Most of the variables above have a 'type'

    A byte is what you make when you don't want anything except the data, format agnostic.

    Let's creating a bytes object, 4 bits long.

    In [46]:
    bytes(4)
    Out[46]:
    b'\x00\x00\x00\x00'

    The b indicates this is a bytes object.

    Each byte is represented by \x, followed by two hexadecimal (base 16) characters

    This means each byte has 256 (16x16) possible states

    But empty bytes data isn't too interesting.

    Create some bytes with data:

    In [47]:
    emojibyte = bytes('💩', 'utf-8') #encoding the poo, from utf-8 into bytes
emojibyte
    Out[47]:
    b'\xf0\x9f\x92\xa9'

    How do we do this the otherway around?

    We can use the decode function:

    In [48]:
    emojibyte.decode('utf-8')
    Out[48]:
    '💩'

    Like tuples, bytes data is immutable, meaning we can't append or modify without completely replacing the contents.

    If we want to be able to more easily modify bytes data, we can convert it to an array.

    In [49]:
    emojibyte = bytearray(emojibyte)
emojibyte
    Out[49]:
    bytearray(b'\xf0\x9f\x92\xa9')

    With the data in this format, we can work on the data much like a string or regular array.

    We can call or modifying a specific value in the array.

    In [50]:
    emojibyte[3] = int('85',16)
emojibyte
    Out[50]:
    bytearray(b'\xf0\x9f\x92\x85')
    In [51]:
    emojibyte.decode('utf-8') #decoding our newly modified array
    Out[51]:
    '💅'

    Sets are unsubscriptable¶

    Since the order of a set is not maintained, using things like the slice function, or append will not work.

    Slice requires an element's position within the set, which makes no sense for an unordered collection of values.

    Similarly 'append' puts a value at the end of a list. Since there is no order, there is no end.

    Modulus¶

    Self and class instances¶

    The speak function appears to not have any arguments. It's an empty parenthesis.

    But if you look up above at when this function is defined, the argument is 'self'

    'my_dog' and 'another_dog' can be considered instances of the class: 'Dog'

    It is self-referencing.

    Classes, Objects, Attributes - the structure¶

    Dog is a CLASS.

    another_dog is an OBJECT, which is one example within the class.

    the variables within the classes, ie. name, legs are called ATTRIBUTES

    Each object is defined by the same list of attributes, for example, each dog might have legs, but the values of each may be different, for example a dog may have 4, 3 or 0 legs.

    speak is a METHOD, which is a function that can be used on any object within that class.

    These methods will only work for objects if they have been defined within this class structure.

    All of this is important to understanding the concepts behind python, which is known as an object-oriented language.

    Threads and Processes¶

    Threads¶

    A thread is a single code line, working its way down the structure.

    We can import multithreading module to allow the computer to run two in parallel. This is useful if some parts of your code have lots of waiting.

    The computer does this by executing a line of code from thread 1, then one from thread 2, then back to the next line in thread 1 etc. If one of the lines isn't ready yet, it keeps executing the line from the process that is still functioning.

    Processes¶

    Multi Processing is kind of similar, but instead of using one processor, we are dedicating a specific processor to each task.

    The catch is that separate processes don't share memory, so they can't see any of the variables from a parallel process.

    This can get weird, because the outputs might not come out predictably if you print them directly.
